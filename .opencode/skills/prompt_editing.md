# Prompt 编辑原则

## 核心原则

1. **只增不减原则** - 修改 prompt 时，只能增加内容，不能删除或大幅减少现有内容
2. **合理规划原则** - 新增内容应该经过合理规划，逻辑清晰，符合整体架构
3. **完整性原则** - 确保修改后 prompt 仍然是完整的、可用的，不会破坏 workflow

## 适用范围

本原则适用于以下目录中的所有 prompt 文件：
- `prompts/` 目录下的所有 Python 文件
- 任何包含 system prompt 或指令文本的文件

## 修改流程

1. **修改前**：
   - 完整阅读现有 prompt 内容
   - 理解当前 prompt 的作用和上下文
   - 确认修改的必要性和目标

2. **修改时**：
   - 使用 Edit 工具进行精确替换
   - 新增内容追加在现有内容之后或适当位置
   - 保留所有现有的指令、示例和格式

3. **修改后**：
   - 检查修改是否影响其他相关 prompt
   - 确保修改后的 prompt 逻辑完整
   - 如有疑问，先向用户确认再执行

## 禁止行为

- 禁止直接替换整个 prompt 内容
- 禁止删除现有指令而不提供替代方案
- 禁止大幅缩减 prompt 长度
- 禁止在未确认的情况下修改核心 workflow 逻辑
